# CheXpert Evaluation Workflow R7

## Status

This report records the addition of a reproducible local evaluation workflow runner for the MedShiftLab-CXR real-data evaluation sequence.

The workflow runner orchestrates the reusable evaluation scripts added during R2-R5:

1. threshold sweep
2. calibrated threshold evaluation
3. bootstrap uncertainty
4. subgroup audit

This is not a full benchmark, not external validation, not fairness validation, and not clinical validation.

## Added workflow script

- `scripts/run_evaluation_workflow.py`

The runner accepts:

- prediction JSON
- labels CSV
- metadata CSV
- subgroup columns
- output directory
- threshold grid settings
- bootstrap settings

It supports two modes:

- default dry-run mode: prints planned commands without executing them
- `--execute` mode: runs the full local evaluation workflow

## Expected input files

The workflow expects a prediction JSON following the MedShiftLab prediction schema, a labels CSV keyed by `sample_id`, and a metadata CSV keyed by `sample_id`.

Minimum metadata schema:

| Column | Purpose |
|---|---|
| `sample_id` | join key |
| user-selected subgroup columns | e.g. `sex`, `view_position`, `age_bucket` |

## Output files

The workflow writes local/private outputs:

| Output | Source step |
|---|---|
| `threshold_sweep.json` | threshold sweep |
| `threshold_sweep.csv` | threshold sweep |
| `threshold_sweep_best.csv` | threshold sweep |
| `calibrated_threshold_eval.json` | calibrated threshold evaluation |
| `calibration_best.csv` | calibrated threshold evaluation |
| `evaluation.csv` | calibrated threshold evaluation |
| `split.csv` | calibrated threshold evaluation |
| `bootstrap_uncertainty.json` | bootstrap uncertainty |
| `bootstrap_uncertainty.csv` | bootstrap uncertainty |
| `subgroup_audit.json` | subgroup audit |
| `subgroup_label_metrics.csv` | subgroup audit |
| `subgroup_aggregate.csv` | subgroup audit |

These output files should stay local/private unless explicitly sanitized.

## Example command

```bash
python3 scripts/run_evaluation_workflow.py \
  --predictions results/local_private_runs/example/predictions.json \
  --labels-csv results/local_private_runs/example/labels.csv \
  --metadata-csv results/local_private_runs/example/metadata.csv \
  --subgroup-columns sex view_position age_bucket \
  --output-dir results/local_private_runs/example/evaluation_workflow \
  --n-bootstrap 1000 \
  --execute
```

Without `--execute`, the runner only prints the planned commands.

## Interpretation

R7 makes the R2-R5 evaluation process reproducible and easier to apply to future datasets or external-validation candidates.

The runner does not change the scientific claim level. It only packages the evaluation workflow. Results generated by this workflow remain exploratory unless they are produced under a proper validation protocol.

## Data handling

Raw data, labels, predictions, split files, bootstrap outputs, subgroup outputs, and local/private workflow results should not be committed directly.

Only sanitized reports and reusable scripts should be committed to GitHub.
